# 학습지원 소프트웨어 개인정보 동의 안내·웹앱·운영 프로세스 — 개인정보 보호법 점검

> **PDF 안내문**: 「학습지원 소프트웨어 개인정보 수집·이용·제공 동의 안내.pdf」는 `개인정보` 폴더에 있습니다. PDF는 바이너리라 자동 추출이 어려우므로, 아래 항목(수집·이용 목적·항목·기간·거부권, 제3자 제공 시 제공받는 자·목적·항목·기간·거부권)이 포함되어 있는지 **직접 확인**하시고, 웹앱·CSV 문구와 일치하는지 점검하시기 바랍니다.

## 1. 운영 프로세스 요약 (이해한 내용)

| 단계 | 주체 | 내용 |
|------|------|------|
| 1 | **담당자(관리자)** | 학운위 심의 파일 업로드 → 학운위 심의를 받은 에듀테크 목록 등록 |
| 2 | **교사** | 위 목록 중 사용할 에듀테크를 양식에 맞춰 작성 → CSV 업로드 |
| 3 | **담당자** | 교사가 올린 목록 확인 → 학운위 심의 여부 점검, 명칭만 다르고 동일 제품이면 「심의 확인」 체크 |
| 4 | **담당자** | 심의 모두 확인 완료 후, 교사가 QR 코드를 가정통신문에 넣어 출력·배부 가능 |
| 5 | **교사** | 가정통신문(QR 포함) 출력 후 학부모에게 배부 |
| 6 | **학부모** | QR 스캔 → 폰으로 에듀테크별 **수집·이용 동의**·**제3자 제공 동의** 체크 → 제출 |
| 7 | **학부모** | 제출 후 가정통신문에 **자필 서명** → 담임에게 제출 |

## 2. 개인정보 보호법(동의 요건) 기준

- **수집·이용 동의** (시행령 제17조 제1항 등):  
  자유의사, **구체성·명확성**, 이해용이성, **동의 여부를 명확히 표시할 수 있는 방법** 제공.
- **제3자 제공 동의**:  
  제공받는 자, 이용 목적, 제공 항목, 보유·이용 기간, **동의 거부권 및 거부 시 불이익** 안내.

(위 요건은 2024년 개정 시행령·해석을 반영한 일반적 기준입니다.)

## 3. 점검 결과 요약

### 3.1 웹앱 자체 (이 시스템에서 수집·이용하는 개인정보)

| 항목 | 점검 내용 | 판단 |
|------|-----------|------|
| 목적 고지 | 학부모 로그인 단계에서 「목적: 학급 에듀테크 활용을 위한 사전 동의 현황 파악 및 확인 코드 생성」 안내 | ✅ 적합 |
| 수집 항목 고지 | 학생 이름, 학번, 학부모 이름, 설정 비밀번호(4자리) 명시 | ✅ 적합 |
| 보유 기간 | 「당해 학년도 종료 시(또는 목적 달성 시) 즉시 파기」 안내 | ✅ 적합 |
| 거부권 안내 | 「거부 시 확인 코드 생성이 불가하여 종이 서면 제출 시 처리가 지연될 수 있음」 명시 | ✅ 적합 |
| 동의 방법 | 정보 입력 후 「동의서 작성하기」로 진행 — 동의 여부를 명확히 표시하는 절차 있음 | ✅ 적합 |

→ **이 웹앱이 직접 수집·이용하는 개인정보(학생명, 학번, 학부모명, PIN, 동의 응답 등)**에 대해서는, 현재 구현만 보면 개인정보 보호법상 **동의 요건을 충족하는 구조**로 보입니다.

### 3.2 에듀테크별 수집·이용·제3자 제공 동의 (학부모 화면)

| 항목 | 점검 내용 | 판단 |
|------|-----------|------|
| 수집·이용 동의 | 에듀테크별로 **수집·이용 동의** 문구(팝업) + 동의/비동의 선택 가능 | ✅ 적합 |
| 제3자 제공 동의 | 에듀테크별로 **제3자 제공 동의** 문구(팝업) + 동의/비동의 선택 가능 | ✅ 적합 |
| 구체성·이해용이성 | CSV로 입력한 「수집이용동의」「제3자제공동의」 문구가 그대로 노출 — **내용이 법령 요구사항(목적, 항목, 기간, 거부권 등)을 담고 있는지는 CSV 작성 내용에 따름** | ⚠️ CSV·가이드에서 필수 기재 항목 안내 필요 |
| 거부권 안내 | 동의서 페이지에 「법적으로 규정된 목적 이외의 수집 항목에 대한 동의를 거부할 권리가 있으며, 거부 시 해당 서비스 제한이 있을 수 있음」 문구 있음 | ✅ 적합 |

→ **에듀테크별 동의**는 “동의 받기 전에 내용을 보여 주고, 동의/비동의를 명확히 선택하게 한다”는 점에서 법적 요건에 부합하는 흐름입니다.  
다만 **실제 법적 유효성**은 각 에듀테크별 문구(수집이용동의·제3자제공동의)가 개인정보 보호법·시행령에서 요구하는 항목(제3자 제공 시 제공받는 자, 목적, 항목, 기간, 거부권 등)을 포함하는지에 따라 달라집니다. 이는 **PDF 안내문·CSV 양식·에듀테크 제공자 안내**에서 보완하는 영역입니다.

### 3.3 운영 프로세스 (담당자 → 교사 → 학부모)

| 단계 | 점검 내용 | 판단 |
|------|-----------|------|
| 학운위 심의 목록 관리 | 담당자가 심의 완료된 에듀테크만 등록·확인 — 교육 과정상 요구되는 절차와 부합 | ✅ 위배 아님 |
| 교사 CSV 업로드 | 교사가 **심의된 목록 안에서** 사용할 에듀테크를 양식에 맞춰 등록 — 수집·이용/제3자 제공 문구를 CSV로 관리 | ✅ 위배 아님 |
| 담당자 심의 확인 | 명칭만 다르고 동일 제품이면 「심의 확인」 처리 — 동의 대상 에듀테크의 적법성 확인 절차로 볼 수 있음 | ✅ 위배 아님 |
| QR·가정통신문 배부 | 학부모가 **QR로 동의 화면에 접근** → **에듀테크별로** 수집·이용·제3자 제공 동의 체크 | ✅ 동의 절차가 “정보주체가 내용을 인지한 후 선택”하는 구조 |
| 자필 서명·제출 | 온라인 제출 후 **가정통신문에 자필 서명**하여 담임에게 제출 — **서면 동의 보강**으로 해석 가능 | ✅ 위배 아님, 오히려 동의 증명 보강 |

→ 현재 설명하신 **전체 프로세스**는, “학운위 심의 → 교사 선택 → 담당자 확인 → 학부모에게 QR·가정통신문 배부 → 학부모가 앱에서 에듀테크별 동의 후 제출 + 자필 서명”이라는 흐름으로 보았을 때 **개인정보 보호법에 반하는 구조로는 보이지 않습니다.**

## 4. 보완 권고 사항 (법 위배 여부보다는 실무 보강)

1. **PDF 안내문**  
   - 「학습지원 소프트웨어 개인정보 수집·이용·제공 동의 안내.pdf」에 **수집·이용 목적, 항목, 보유 기간, 거부권**, 제3자 제공 시 **제공받는 자, 목적, 항목, 기간, 거부권**이 포함되어 있는지 학교·법무 담당자가 한 번 더 확인하는 것이 좋습니다.

2. **CSV 양식·가이드**  
   - 「수집이용동의」「제3자제공동의」 칸에 **개인정보 보호법·시행령에서 요구하는 필수 기재 항목**을 포함하도록 양식 설명 또는 예시 문구를 넣으면, 에듀테크별 동의 문구의 법적 완결성이 높아집니다.

3. **보유 기간·파기**  
   - 웹앱에 안내한 “당해 학년도 종료 시(또는 목적 달성 시) 즉시 파기”를 **실제 파기 절차(DB·백업 삭제 등)**와 매칭해 운영하는지 정기적으로 점검하는 것을 권장합니다.

4. **재동의 사유**  
   - 에듀테크 목록·수집 항목·제3자 제공 내용 등이 바뀌면 **변경 사항 안내 후 재동의**를 받는 절차를 두는 것이 법령 해석상 안전합니다.
